# 소프트웨어 요구사항 명세서 (SRS)

**프로젝트명:** 지하차도 침수 위험 예측 시스템  
**버전:** 1.0  
**작성일:** 2026-06-29  

---

## 1. 개요

### 1.1 목적
본 문서는 머신러닝 기반 지하차도 침수 위험 예측 시스템의 기능적·비기능적 요구사항을 정의한다.

### 1.2 범위
- 강수량 및 지하차도 물리 정보를 입력받아 침수 위험도를 예측하는 웹 시스템
- 전국 주요 지하차도 침수 사례를 지도로 시각화
- 예측 결과를 3단계(낮음 / 중간 / 높음)로 표시

### 1.3 용어 정의

| 용어 | 설명 |
|------|------|
| 침수 위험도 | 현재 강수 조건에서 해당 지하차도가 침수될 확률 |
| ERA5 | 유럽 중기예보센터(ECMWF)의 위성 기반 기상 재분석 데이터 |
| AUC | 모델 분류 성능 지표 (1에 가까울수록 우수) |
| 호우경보 기준 | 3시간 강수량 90mm 이상 또는 12시간 강수량 180mm 이상 |

## 2. 전체 시스템 설명

### 2.1 시스템 구성

```
사용자 (브라우저)
    ↕ HTTP
Flask API 서버 (app.py)
    ↕ pickle
ML 모델 (Random Forest / XGBoost)

```

### 2.2 사용 기술

| 구분 | 기술 |
|------|------|

| 백엔드 | Python 3.x, Flask 3.0 |
| 프론트엔드 | HTML5, JavaScript, Leaflet.js |
| 데이터 | 공공데이터포털 지하차도 현황 CSV + Open-Meteo ERA5 강수량 |

### 2.3 사용자 대상
- 지자체 재난안전 담당자
- 기상 상황 모니터링 운영자
- 해커톤 심사위원 및 일반 시연 관람자

## 3. 기능 요구사항

### FR-01. 침수 위험도 예측
- **설명:** 사용자가 강수량과 지하차도 정보를 입력하면 침수 확률(%)과 위험 등급을 반환한다
- **입력값:**
  - 시간당 강수량 (mm/h)
  - 3시간 누적 강수량 (mm)
  - 6시간 누적 강수량 (mm)
  - 24시간 누적 강수량 (mm)
  - 지하차도 깊이 (m)
  - 지하차도 길이 (m)
  - 준공연도
  - 차로수
  - 배수펌프 유무 (있음/없음)
  - 하천 인접 여부 (인접/해당없음)
- **출력값:**
  - 침수 확률 (0.0 ~ 100.0 %)
  - 위험 등급 (낮음 / 중간 / 높음)
  - 위험도 색상 코드 (초록 / 주황 / 빨강)
- **위험 등급 기준:**
  - 낮음: 확률 0 ~ 39%
  - 중간: 확률 40 ~ 69%
  - 높음: 확률 70% 이상

### FR-02. 침수 사례 지도 시각화
- **설명:** 전국 주요 지하차도 침수 사례를 Leaflet.js 지도 위에 마커로 표시한다
- **마커 구분:**
  - 빨간색 (대형 마커): 인명피해 발생 사례
  - 주황색 (소형 마커): 재산피해만 발생한 사례
- **클릭 시 표시:** 지하차도명, 침수연도, 피해 유형

### FR-03. 예측 API 제공
- **설명:** `/predict` 엔드포인트로 JSON 요청을 받아 예측 결과를 JSON으로 반환한다
- **Method:** POST
- **Request Body:** (JSON)
```json
{
  "rainfall_1h": 35,
  "rainfall_3h": 80,
  "rainfall_6h": 130,
  "rainfall_24h": 180,
  "depth_m": 4.5,
  "length_m": 300,
  "built_year": 1998,
  "lanes": 4,
  "has_drainage_pump": 0,
  "nearby_river": 1
}
```
- **Response Body:** (JSON)
```json
{
  "probability": 0.8231,
  "percent": 82.3,
  "risk_level": "높음",
  "color": "#e74c3c",
  "model": "XGBoost"
}
```

### FR-04. 모델 학습 스크립트
- **설명:** `train_model.py` 실행 시 데이터를 불러와 모델을 학습하고 `model/` 폴더에 저장한다
- **학습 모델:** Logistic Regression, Random Forest, XGBoost
- **저장 파일:** `best_model.pkl`, `scaler.pkl`, `features.pkl`
- **자동 선택:** 테스트 AUC 기준 최고 성능 모델을 `best_model.pkl`로 저장

## 4. 비기능 요구사항

### NFR-01. 성능
- 예측 응답 시간: 1초 이내

## 6. 제약 사항

- ERA5 재분석 데이터는 약 9km 격자 단위로 제공되어 국지성 집중호우는 과소평가될 수 있음
- 침수 사례 60건 중 일부는 정확한 침수 날짜를 확인하지 못해 주요 집중호우 발생일로 대체함
- 배수펌프 유무·하천 인접 여부는 실측 데이터가 부족하여 통계적 추정값 사용
